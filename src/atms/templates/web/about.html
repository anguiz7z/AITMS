About ATMS

AI Threat Modeling Studio is a workflow tool for threat-modeling AI/ML/LLM/agentic systems against:

  • OWASP LLM Top 10 (2025) — the canonical risk list for LLM applications.
  • OWASP Agentic AI — Threats and Mitigations (OWASP ASI, 2025) — 15 canonical threats (T1–T15) specific to agentic AI, plus 2 ATMS extensions,, including memory poisoning, tool misuse, intent breaking, rogue agents, and inter-agent protocol abuse. Browse →
  • CSA MAESTRO (2025) — the 7-layer agentic-AI threat-modeling reference architecture (Foundation Models / Data Operations / Agent Frameworks / Deployment & Infrastructure / Evaluation & Observability / Security & Compliance / Agent Ecosystem). Browse →
  • MITRE ATLAS — adversarial tactics, techniques, and case studies for ML/AI systems.
  • NIST AI RMF (AI 600-1) GenAI Profile — risk-management framework for GenAI.
  • STRIDE for AI — Microsoft's STRIDE applied to AI/ML primitives. AI-specific subcategories live in kb/stride_ai_matrix.yaml; this is our extension of STRIDE, not a peer-reviewed standalone methodology.
  • Likelihood × Impact (DREAD-derived) — risk score on a 0–100 scale, computed from a 1–5 likelihood and 1–5 impact with weighting borrowed from Microsoft's deprecated DREAD model. Earlier ATMS versions called this "DREAD-AI"; that was misleading — there is no published "DREAD-AI" standard.
  • Singapore CSA Guidelines on Securing AI Systems (2024) — cross-walked into per-component playbooks for regional regulatory alignment.

What ATMS is — and isn't

It is an AI-induced-risk evaluator: it considers the full architecture an AI sits in (firewalls, databases, network switches, OT, mainframes — everything) but only emits threats that exist because of the AI integration. A banking ATM with no LLM produces zero threats; the same ATM with an LLM-based fraud-detection sidecar produces threats specific to that integration, with provenance pointing at the AI component that creates each risk.

It isn't a general-purpose threat modeler. A system with zero AI components is rejected at load time with a clear message. For pure-IT systems use OWASP Threat Dragon, Microsoft Threat Modeling Tool, or IriusRisk.

Threat-category provenance

ATMS tags every threat with one or more "STRIDE for AI" categories. Six come straight from Microsoft's 1999 STRIDE; three are ATMS extensions anchored to other published frameworks (MITRE ATT&CK, NIST AI RMF, MITRE ATLAS). Design choices

  • Deterministic core. No LLM call is required to run an analysis. Threats come from curated per-component playbooks. Mappings to ATLAS and OWASP are explicit, not inferred.
  • Optional vision. If you set ANTHROPIC_API_KEY, ATMS can parse architecture-diagram images via Claude vision and emit a draft system YAML. With no key, you describe systems in YAML manually.
  • Workflow-first. CLI and minimal web UI. No multi-tenant auth, no DB, no cloud dependency. Run it on your laptop.
  • Open framework data. All KB entries cite their source. You can fork the playbooks to fit your environment.

Disclaimer

ATMS is decision support — its output is a starting point for a threat model, not an authoritative security assessment. Have a security professional review the report before treating any finding as actionable.
