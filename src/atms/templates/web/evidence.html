What ATMS does with evidence

  1. Parse — Nessus XML, SARIF (CodeQL / Semgrep / Trivy / Bandit), generic CSV (auto-sniffs CVE / Severity / Asset columns), or STIX 2.1 bundles.
  2. Match — finding → component by hostname, IP, FQDN, product+version, vendor token, or component-name substring.
  3. Decorate — every CVE is checked against the bundled CISA KEV catalog and EPSS top-N snapshot; matching evidence is flagged.
  4. Apply — KEV → severity = critical, status = exploited; high-severity scanner finding → status = observed, likelihood + 1; TI hit → status = likely; red-team result → status = exploited, likelihood = 5.
  5. Render — reports show evidence rows under each threat, a KEV badge column, and an evidence-status breakdown (hypothetical | likely | observed | exploited).

Refresh the bundled KEV / EPSS snapshots with atms refresh-feeds (planned for v0.13). For now the bundled snapshot is current as of 2026-05-10 and covers ~120 CVEs intersecting the device catalog's products.
