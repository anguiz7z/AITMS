Infrastructure-as-Code ingest docker-compose or Terraform → draft System YAML

Upload a docker-compose.yml or a Terraform .tf file and ATMS produces a draft System YAML that you can review in the editor before running analysis. Service / resource types are heuristically mapped to ATMS component types; vendor / image / version travel into component.metadata so the device-catalog and evidence matchers light up immediately.

What ATMS recognises

  • docker-compose: services → components, networks → trust zones, depends_on → dataflows, host-mapped ports → user-facing edge. Vendor / image / version sniffed from the image tag.
  • Terraform: AWS / Azure / GCP resource types → components. depends_on + cross-resource interpolations → dataflows. Modules + count/for_each may need a terraform show -json expansion first.

After conversion, drop the YAML into the Editor to fix anything classified as other and to assign a vendor / product / version per component (which lets the v0.12 evidence matcher light up KEV CVEs automatically).
