Upload a diagram (.drawio, .mermaid, .vsdx, .md, .png, .jpg)

Upload an existing architecture diagram and ATMS will produce a draft System YAML in the box below — components, dataflows and trust boundaries auto-classified — review and edit before clicking Analyze.

  • .drawio / .xml (diagrams.net): deterministic. 110+ AWS / Azure / GCP stencil prefixes auto-recognised; VPC / subnet / DMZ containers become trust boundaries; crossings auto-flagged on dataflows. (v0.17.4)
  • .mmd / .mermaid / .md (Mermaid flowchart): deterministic. Markdown files have their first ```mermaid fence extracted. Shape semantics encoded (cylinder→database, circle→user, hexagon→agent, …); subgraphs labelled VPC/DMZ/etc. become trust boundaries. (new in v0.18.1)
  • .vsdx (Visio): deterministic. Extracts shapes + connectors, classifies by keyword. (Legacy .vsd not supported — re-save as .vsdx in Visio first.)
  • .png / .jpg / .webp: opt-in vision. Requires ANTHROPIC_API_KEY + the [vision] extra. The shipped .exe doesn't bundle the vision module — install with pip install atms[vision] if you need image ingestion.

System YAML

AI-native (full pipeline): enumerates AI-specific threats per component, mapped to OWASP LLM Top 10, OWASP Agentic (T1–T15), MITRE ATLAS, CSA MAESTRO & the Singapore CSA AI guidelines — then classified with extended STRIDE-for-AI categories and cross-referenced to OWASP API, ATT&CK Cloud/Enterprise, LINDDUN & NIST AI 100-2. PASTA: keep only threats in attack paths or with likelihood ≥ 4. LINDDUN: filter to privacy-only threats.

Tip: editing raw YAML is fragile (typing IoT Device instead of iot_device will reject the analysis). The visual editor uses dropdowns so you can't pick an invalid component type.

What ATMS does

  1. Identifies AI primary components and traces the dataflow blast radius — non-AI components reached by an AI primary become "AI-adjacent" and pick up provenance-tagged threats. Components outside the radius emit none.
  2. Emits structured threats with STRIDE for AI, OWASP LLM Top 10, OWASP Agentic AI (T1–T15), MITRE ATLAS, CSA MAESTRO, and Singapore CSA Guidelines — no hallucinations.
  3. Builds an attack-path graph honoring ATLAS tactic ordering.
  4. Scores each threat (likelihood × impact, DREAD-derived 0–100 scale).
  5. Recommends mitigations from ATLAS Mitigations + OWASP LLM + OWASP Agentic + inline.
  6. Exports Markdown, HTML, STIX 2.1, ATLAS Navigator JSON, and CSV.

No LLM call is required — this is deterministic Python over a curated knowledge base. Optional vision-based diagram parsing can be enabled if you set ANTHROPIC_API_KEY.
